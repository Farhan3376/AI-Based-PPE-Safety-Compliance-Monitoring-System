# 👷‍♂️ AI-Based PPE Safety Compliance Monitoring System

An end-to-end multimodal computer vision pipeline built for Google Colab that automates industrial safety audits. This system leverages state-of-the-art AI models to monitor worker Personal Protective Equipment (PPE) compliance, track worker movement, and generate structured reasoning reports.

## 🚀 Features

- **High-Speed Object Detection**: Uses the latest **YOLO26** (`yolo26n.pt`) for end-to-end, NMS-free detection of workers and PPE items (helmets, vests, boots, etc.).
- **Persistent Worker Tracking**: Integrates **ByteTrack** to assign and maintain unique IDs for workers across video frames, allowing for accurate trajectory mapping.
- **Multimodal AI Reasoning**: Utilizes the **Google Gemini 2.0 Flash API** to perform zero-shot temporal safety analysis on the video, identifying missing PPE and generating structured JSON reports.
- **Trajectory Mapping**: Automatically plots 2D scatter-plot radar maps of worker movements to identify restricted area entry or anomalous behavior.
- **Standalone Notebook Architecture**: Fully encapsulated in a single `.ipynb` Jupyter Notebook designed to run seamlessly in Google Colab.

## 🛠️ Tech Stack

- **Computer Vision**: Ultralytics YOLO26, OpenCV (`opencv-python-headless`)
- **Tracking**: ByteTrack (`persist=True` built into Ultralytics tracker)
- **Multimodal LLM**: Google GenAI SDK (`gemini-2.0-flash`)
- **Data & Visualization**: Pandas, NumPy, Matplotlib

## 💻 How to Run (Google Colab)

1. **Open the Notebook**: Upload `PPE_Project.ipynb` to Google Colab.
2. **Upload Video**: In the Colab file explorer, create an `input` folder and upload your test video as `input/original_video.mp4`.
3. **Configure API Key**: Locate **Cell 3** in the notebook and insert your Gemini API Key:
   ```python
   GEMINI_API_KEY = "YOUR_GEMINI_API_KEY_HERE"
   ```
4. **Run All**: Go to `Runtime > Run all` to execute the pipeline. 

*(Note: The system extracts frames, applies Gaussian blur preprocessing, runs detection/tracking, visualizes bounding boxes, and uploads the video to Gemini for final reasoning).*

## 📊 Results & Outputs

The system saves all analysis results into specific output directories (e.g., `output2/` or `output3/` depending on your configuration). Each directory contains:

1. **`output_detection.mp4`**: The processed video featuring thick, readable bounding boxes, confidence scores, object class names, and tracking IDs.
2. **`coordinates.csv`**: Frame-by-frame (X, Y) pixel coordinates for every tracked worker ID.
3. **`radar_map.png`**: A 2D visualization plotting the trajectories of all workers across the site layout.
4. **`output_log.json`**: A highly structured JSON report generated by Gemini, outlining:
   - Overall Site Safety Status
   - Number of Helmet/Vest Violations
